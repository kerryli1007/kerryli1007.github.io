Q3_Week3_工作总结

Posted on 2020-08-02 | In 工作

原则：以周为粒度，总结工作，从中提炼规律，沉淀属于自己的方法论。

如何做好售前

背景：AI项目，从项目的生命周期上来讲，一共分如下几步

销售阶段是销售找到线索，根据客户的潜在需求，进行挖掘，让客户意识到，我们可以解决对方的问题。

售前阶段是尽可能的收集客户的业务信息，这里的尽可能地收集客户的业务信息是什么意思呢？就是说，收集到的信息，足够我们根据客户的业务痛点，提供解决方案。

项目经理阶段是基于售前收集充分的信息，制定解决方案，协调资源，推动项目落地。

解释完三个阶段的工作内容。我之前的工作，更偏向于项目经理阶段，就是当售前同事，已经将基本信息都收集的差不多，这个时候，我介入。在心态上，我一直都以为，售前其实很好做的，但是最近，我开始正式做售前了～我发现这事没有我原来想得那么简单。为什么呢？

首先，售前需要客户业务是否足够了解，往上一层，是需要对客户行业都有足够的了解。只有当你足够了解，当你和客户沟通，自诩“我是来帮你解决问题的”，对方才会买账。但是，对行业的了解，对客户业务的了解，都非常依赖经验。而我刚好，对广告CTR行业的经验少，准确地说，我对收集哪些信息，可以辅助判断我们是否可以解决该客户的问题，把控不够。这个把控不够，体现在

1）我不知道该收集哪些信息
2）就算我会前，做了准备，但是在会上沟通时候，我的沟通思路不够清晰，逻辑不够严谨

所以，如何逐个突破这些点呢？

这是我根据做的几个项目总结出来的CTR售前框架。

根据这个框架，我们来复盘一下今天和jc CTO的沟通。

重新Set Up一下背景，客户的主要业务是互动广告，关于互动广告是什么，可自行百度。

我们了解的第一个问题是

当前的规则是什么？

客户回答：“当前的投放，基本上就是广告渠道和广告主的匹配”。

理解一下，这个回答，等于是说，比如来了一个广告主，是电商属性的，我们找到一个渠道，觉得这个渠道适合电商广告主，我们就推广投放。

但是这个规则还是比较粗，我们更好奇说，那这种匹配规则，是否是已经写在系统中，比如电商属性广告主我们就投A渠道。

接着客户回答：“我们是动态调整的，按天为维度，做调整，如果今天A渠道的表现不好，那么我们会为广告主切B渠道。”

这是什么意思呢？就是现在还不是竞价机制，就是买断一个渠道，如果这个渠道在一段时间内表现不好，是可以切换到其他渠道的。这也是客户觉得当前的一个痛点，就是说，这种投放机制下，一个渠道的用户，看到的广告，都是相同的，没有办法千人千面。

看吧，这些细节，都是在不断的对话过程中，才挖掘出来的。需要售前对业务足够熟悉，也足够敏感，在关键的地方提问。

了解当前规则之后，很自然，我们会好奇这种规则下的表现对吧？因此，就来到我们的第二个问题。

当前基于规则的投放，投放效果如何？

需要注意哦，投放效果的定义需要先和对方明确，是点击率、还是下载率、还是注册率。这里又有一个和业务相关的点，因为客户是互动广告，所以根据广告投放类型不同，转化效果其实不同的。比如有优惠券领取广告，是需要填手机号码领取优惠券，也有导购广告，是看是否有成单。

说到投放效果，就会说到，反馈数据，这种互动广告的反馈数据是如何的呢？客户分享到，这种是广告主会给反馈，比如某个时间点，某个用户有一个下载，或者注册。这就是反馈数据的构成。谈到这里，其实也就差不多了。

最后，我们就来看看数据情况啦，一般来说，广告CTR字段，我们是希望数据字段越多越好。

这些字段又分为2个维度，一个是用户维度，一个是广告特征。

所以我们问出了第三个问题

咱们现在支持的字段有哪些呀？

客户也给了回答，字段挺少的。

复盘到此结束，为什么我上午会觉得很乱糟糟呢？因为没有结构，这里问一下，那里问一下，不单单我自己觉得很混乱，客户也觉得很混乱。

机器学习指标

最近做广告CTR项目，重温了一下指标。

先了解阈值、混淆矩阵、接着，我们了解一下准确率、召回率、精确率，最后我们来看看AUC。

阈值

逻辑回归返回的是概率。

如果某个逻辑回归模型对某封电子邮件进行预测时返回的概率为 0.9995，则表示该模型预测这封邮件非常可能是垃圾邮件。相反，在同一个逻辑回归模型中预测分数为 0.0003 的另一封电子邮件很可能不是垃圾邮件。可如果某封电子邮件的预测分数为 0.6 呢？为了将逻辑回归值映射到二元类别，您必须指定分类阈值（也称为判定阈值）。如果值高于该阈值，则表示“垃圾邮件”；如果值低于该阈值，则表示“非垃圾邮件”。

人们往往会认为分类阈值应始终为 0.5，但阈值取决于具体问题，因此您必须对其进行调整。

阳性和阴性：正分类与负分类

我们借用一个狼来了的故事，来总结我们的“狼预测”模型，使用混淆矩阵描述所有可能出现的结果
(img)
真正例是指模型将正类别样本正确地预测为正类别。同样，真负例是指模型将负类别样本正确地预测为负类别。
假正例是指模型将负类别样本错误地预测为正类别，而假负例是指模型将正类别样本错误地预测为负类别。

准确率

准确率是一个用于评估分类模型的指标。通俗来说，准确率是指我们的模型预测正确的结果所占的比例。

准确率 = 预测正确的样本数量 / 被预测的样本总数

Accuracy = (TP + TN) / (TP + TN + FP + FN)
其中 TP = 真正例，TN = 真负例，FP = 假正例，FN = 假负例

让我们来试着计算一下以下模型的准确率，该模型将 100 个肿瘤分为 恶性 （正类别）或 良性 （负类别）：

Accuracy = (1 + 90) / (1 + 1 + 8 + 90) = 0.91

准确率为 0.91，91%（总共100个样本中有91个预测正确）。这是否是表明我们的肿瘤在识别恶性肿瘤方面表现非常出色？

实际上，只要我们仔细分析一下正类别和负类别，就可以更好地了解我们模型的效果。

在 100 个肿瘤样本中，91 个为良性（90 个 TN 和 1 个 FP），9 个为恶性（1 个 TP 和 8 个 FN）。

在 91 个良性肿瘤中，该模型将 90 个正确识别为良性。这很好。不过，在 9 个恶性肿瘤中，该模型仅将 1 个正确识别为恶性。这是多么可怕的结果！9 个恶性肿瘤中有 8 个未被诊断出来！

虽然 91% 的准确率可能乍一看还不错，但如果另一个肿瘤分类器模型总是预测良性，那么这个模型使用我们的样本进行预测也会实现相同的准确率（100 个中有 91 个预测正确）。换言之，我们的模型与那些没有预测能力来区分恶性肿瘤和良性肿瘤的模型差不多。

当您使用分类不平衡的数据集（比如正类别标签和负类别标签的数量之间存在明显差异）时，单单准确率一项并不能反映全面情况。

Recall & Precision
精确率尝试回答的问题是：所有被识别为正的样本中，正确识别为正的样本占比是多少？

数学意义上，精确率的定义Precision = TP / （TP + FP）

该模型的精确率为 0.5，也就是说，该模型在预测恶性肿瘤方面的正确率是 50%。

召回尝试回答的问题是：所有正样本中，我们识别出来的正样本有多少？

数学意义上，召回率的定义 Recall = TP / （TP + FN）

该模型的召回率是 0.11，也就是说，该模型能够正确识别出所有恶性肿瘤的百分比是 11%。

  • 真正例率
  • 假正例率
